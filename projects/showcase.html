Showcase v4 is the fourth iteration of my showcase site, featuring many improvements, both ones visible to you and ones behind the scenes, allowing me to better maintain this site, while featuring more information about each project that is being showcased. Showcase v4 has had refactors for much of the codebase that was ported from v3, new additions not seen before in v3, as well as a complete visual overhaul.

Showcase v4 is availible for free, on GitHub.

Showcase v4 was made with:
  • Languages: HTML 5, CSS and JavaScript, which is sometimes shortened to 'JS'.
  • Engine/Framework: Bootstrap 5.3, a front end framework originally developed for twitter.

Design was thought up by:
  • Ariel / CalicoCatio
The design from v4 is visually very similar to v3, down to the position of the carousels inside each showcase. Behind the scenes, however, the site is radically different. v3 had a focus on making everything as custom as possible, even if it meant a lesser experience for the user. v4 is designed to instead make use of bootstrap as much as possible, only using custom classes and CSS styles when absolutly necessary. This may seem like a bad idea, as if bootstrap goes down, v4 will practiacly be unusable, but bootstrap is open-source, and in use by some of the biggest websites in the world. Spotify, LinkedIn, Twitter, Duolingo, Lyft all use bootstrap for their websites, therefore if it is good enough for them, it is also good enough for me.

Code was written by:

Art was drawn up by:
  • Ariel / CalicoCatio
The only real 'art' in v4 comes as a port from v2, the orange that I use as a favicon. I made this in Paint 3D by using a touchscreen to draw the A over and over again, until I got one that looked acceptable. Since then, the favicon has recived a little TLC as the move from v2 to v3 caused a bit of the image to get cropped off; with v4, I have re-added the missing pixels.

  • When porting some of the CSS from v3 to v4 I noticed that one of the styles restricted the height of the body element, to 100%, or one screen height. This caused a lot of problems when I was working on v3, and I somehow never noticed it.
